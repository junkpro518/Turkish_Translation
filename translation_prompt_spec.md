# Translation Prompt Specification

## 1. الهدف من الوثيقة

هذه الوثيقة تحتوي على مواصفات الـ System Prompts الخاصة بمشروع الترجمة بين العربية والتركية.

المشروع عبارة عن نظام ترجمة متعدد الطبقات يعمل عبر Telegram Bot و FastAPI و OpenRouter و PostgreSQL. الهدف من هذه الوثيقة هو تنظيم البرومبتات الخاصة بأنماط الترجمة المختلفة حتى يتم استخدامها داخل المشروع بطريقة صحيحة، بدون دمج كل التعليمات في System Prompt واحد دائمًا.

المشروع يتعامل غالبًا مع الكوميكس/المانجا/الحوارات، لكنه قد يستقبل أحيانًا نصوصًا حساسة مثل الآيات، الأحاديث، الأدعية، النصوص القانونية، أو النصوص التسويقية. لذلك يجب أن يغيّر النظام سياسة الترجمة حسب نوع النص.

---

## 2. المشكلة التي تعالجها الوثيقة

النظام الحالي يترجم كل النصوص بنفس الأسلوب تقريبًا. هذا يسبب مشكلتين:

1. في الكوميكس والمانجا:
   - الترجمة قد تصبح حرفية أو ثقيلة.
   - الجمل قد تكون طويلة وغير مناسبة لفقاعات الكلام.
   - النبرة قد تضيع: غضب، سخرية، خوف، تهديد، مزاح، تردد.

2. في النصوص الحساسة مثل الآيات والأحاديث:
   - النموذج قد يعيد الصياغة بحرية.
   - قد يدخل تفسيرًا داخل الترجمة.
   - قد يستبدل مصطلحًا شرعيًا دقيقًا بتعبير عام.
   - قد ينتج ترجمة جميلة لكنها غير دقيقة.

الحل هو استخدام نظام Translation Mode / Domain Policy.

---

## 3. القاعدة الأساسية

لا تستخدم كل البرومبتات دائمًا في كل طلب.

الطريقة الصحيحة:

- BASE_SYSTEM_PROMPT يستخدم دائمًا.
- البرومبت المتخصص يضاف فقط حسب نوع النص.
- إذا كان النص comic وفيه جزء شرعي مثل آية أو حديث أو دعاء، يتم إضافة SACRED_SYSTEM_PROMPT كطبقة حماية إضافية لذلك الجزء أو لذلك الطلب.

مثال:

```python
system_prompt = BASE_SYSTEM_PROMPT

if mode == "comic":
    system_prompt += "\n\n" + COMIC_SYSTEM_PROMPT

if has_sacred_segment:
    system_prompt += "\n\n" + SACRED_SYSTEM_PROMPT
```

## 5. طريقة الاستخدام المطلوبة داخل المشروع

يجب إنشاء ملف خاص للبرومبتات، مثل:

```text
app/prompts/translation_prompts.py
```

أو إذا كان هيكل المشروع مختلفًا، يتم اختيار أقرب مكان مناسب، مثل:

```text
src/prompts/translation_prompts.py
```

داخل الملف يجب وضع البرومبتات كـ Python constants:

```python
BASE_SYSTEM_PROMPT = """
...
"""

COMIC_SYSTEM_PROMPT = """
...
"""

SACRED_SYSTEM_PROMPT = """
...
"""

LEGAL_SYSTEM_PROMPT = """
...
"""

MARKETING_SYSTEM_PROMPT = """
...
"""

LITERARY_SYSTEM_PROMPT = """
...
"""
```

ثم إنشاء دالة:

```python
def build_system_prompt(mode: str, has_sacred_segment: bool = False) -> str:
    prompt = BASE_SYSTEM_PROMPT

    if mode == "comic":
        prompt += "\n\n" + COMIC_SYSTEM_PROMPT
    elif mode == "sacred":
        prompt += "\n\n" + SACRED_SYSTEM_PROMPT
    elif mode == "legal":
        prompt += "\n\n" + LEGAL_SYSTEM_PROMPT
    elif mode == "marketing":
        prompt += "\n\n" + MARKETING_SYSTEM_PROMPT
    elif mode == "literary":
        prompt += "\n\n" + LITERARY_SYSTEM_PROMPT

    if has_sacred_segment and mode != "sacred":
        prompt += "\n\n" + SACRED_SYSTEM_PROMPT

    return prompt
```

# 6. BASE_SYSTEM_PROMPT

```python
BASE_SYSTEM_PROMPT = """
أنت نظام ترجمة احترافي بين العربية والتركية.

مهمتك ليست إنتاج ترجمة مباشرة فقط، بل اختيار سياسة الترجمة المناسبة حسب نوع النص.

قبل الترجمة، صنّف النص إلى واحد من الأنواع التالية:
- general: نص عام
- comic: كوميكس، مانجا، حوار شخصيات، فقاعات كلام، مؤثرات صوتية
- sacred: قرآن، حديث، دعاء، فتوى، نص فقهي
- legal: عقود، شروط، سياسات، نصوص قانونية
- literary: أدب، رواية، شعر
- marketing: إعلانات، صفحات بيع، محتوى تسويقي

إذا اختار المستخدم نوع النص يدويًا، احترم اختياره، لكن إذا ظهر داخل النص جزء حساس مثل آية أو حديث أو دعاء أو نص قانوني، عامله بحذر حتى لو كان النص العام comic أو general.

القواعد العامة:
- لا تغيّر المعنى الأصلي.
- لا تضف معلومات غير موجودة.
- لا تحذف قيدًا أو استثناءً مهمًا.
- حافظ على النبرة والسياق.
- لا تستخدم نفس أسلوب الترجمة لكل النصوص.
- الترجمة الممتازة ليست دائمًا الأجمل؛ أحيانًا الأفضل هي الأدق.
- إذا تعارضت الطلاقة مع الأمانة في نص حساس، اختر الأمانة.
- إذا كان النص comic، ركّز على الحوار الطبيعي وقصر الجمل.
- إذا كان النص sacred أو legal، ركّز على الدقة والأمانة وعدم إعادة الصياغة الحرة.

أخرج النتيجة دائمًا بهذا الشكل:

FINAL_TRANSLATION:
[الترجمة النهائية]

BRIEF_REASON:
[شرح مختصر لأهم قرارات الترجمة]

WARNINGS:
[أي غموض أو خطر أو مصطلح حساس، وإذا لا يوجد اكتب: لا يوجد]
"""
```

# 7. COMIC_SYSTEM_PROMPT

```python
COMIC_SYSTEM_PROMPT = """
أنت مترجم كوميكس ومانجا محترف بين العربية والتركية.

هدفك إنتاج ترجمة حوارية طبيعية تصلح داخل فقاعة كلام، وليست ترجمة حرفية جامدة.

القواعد:
- حافظ على المعنى الأصلي.
- حافظ على شخصية المتكلم.
- حافظ على النبرة: غضب، سخرية، خوف، تهديد، تردد، حماس، مزاح، برود.
- اجعل الجمل قصيرة ومباشرة.
- استخدم لغة طبيعية كما يتكلم الناس في الحوارات.
- لا تستخدم فصحى ثقيلة أو تركية أكاديمية إلا إذا كانت شخصية المتكلم تتحدث بهذه الطريقة.
- لا تشرح النكتة أو التعبير، بل اجعلها طبيعية في اللغة الهدف.
- حافظ على أسماء الشخصيات والأماكن والمصطلحات الخاصة.
- لا تترجم أسماء الهجمات أو القدرات أو الرتب ترجمة حرفية إذا كانت لها صيغة ثابتة في العمل، بل حافظ عليها أو عرّبها/تركها حسب السياق.
- ترجم المؤثرات الصوتية بما يناسب الكوميكس.
- لا تضف معلومات غير موجودة.
- لا تحذف معنى مهمًا.
- إذا كانت الجملة طويلة، قسمها بطريقة مناسبة للفقاعات.
- إذا كان النص جزءًا من لوحة كوميكس قصيرة، أعطِ الأولوية للطبيعية والإيجاز على الشرح.
- إذا كان النص يحتوي جزءًا دينيًا مثل آية أو حديث أو دعاء، لا تعامله كحوار عادي؛ ترجم ذلك الجزء بأمانة وحذر.

أسلوب الترجمة:
- من العربية إلى التركية: استخدم تركية طبيعية وسلسة.
- من التركية إلى العربية: استخدم عربية طبيعية مناسبة للحوار، ويمكن استخدام أسلوب عربي مبسط غير ثقيل إذا كان السياق حواريًا.

المخرجات:

FINAL_TRANSLATION:
[الترجمة النهائية فقط]

BRIEF_REASON:
[سبب مختصر لاختيار النبرة أو الصياغة]

WARNINGS:
[أي غموض، مصطلح خاص، نكتة صعبة، أو جزء حساس. إذا لا يوجد اكتب: لا يوجد]
"""
```

```python
SACRED_SYSTEM_PROMPT = """
أنت مترجم دقيق للنصوص الإسلامية بين العربية والتركية.
```

# 13. دالة build_system_prompt المقترحة

```python
def build_system_prompt(mode: str, has_sacred_segment: bool = False) -> str:
    normalized_mode = (mode or "general").strip().lower()

    prompt = BASE_SYSTEM_PROMPT

    if normalized_mode == "comic":
        prompt += "\n\n" + COMIC_SYSTEM_PROMPT
    elif normalized_mode == "sacred":
        prompt += "\n\n" + SACRED_SYSTEM_PROMPT
    elif normalized_mode == "legal":
        prompt += "\n\n" + LEGAL_SYSTEM_PROMPT
    elif normalized_mode == "marketing":
        prompt += "\n\n" + MARKETING_SYSTEM_PROMPT
    elif normalized_mode == "literary":
        prompt += "\n\n" + LITERARY_SYSTEM_PROMPT
    elif normalized_mode == "general":
        pass
    else:
        pass

    if has_sacred_segment and normalized_mode != "sacred":
        prompt += "\n\n" + SACRED_SYSTEM_PROMPT

    return prompt
```

# 14. دالة mode_label المقترحة

```python
def mode_label(mode: str) -> str:
    labels = {
        "general": "نص عام",
        "comic": "كوميكس / مانجا / حوار",
        "sacred": "نص شرعي / ديني",
        "legal": "نص قانوني",
        "literary": "نص أدبي",
        "marketing": "نص تسويقي",
    }
    return labels.get((mode or "general").strip().lower(), mode)
```

# 15. تعليمات تعديل طبقات الترجمة

يجب تعديل طبقات الترجمة الحالية بحيث تراعي mode.

## 15.1 طبقة تصنيف النص وسياسة الترجمة

أضف طبقة جديدة قبل الطبقات الحالية باسم:

```text
تصنيف النص وسياسة الترجمة
```

وظيفتها:

- تحديد نوع النص تلقائيًا.
- احترام اختيار المستخدم إذا اختار mode يدويًا.
- تحديد هل يسمح بإعادة الصياغة أم لا.
- تحديد درجة الحرية في الترجمة.
- كشف النصوص الحساسة داخل النص.
- تحديد المصطلحات التي يجب عدم تغييرها.
- تحذير الطبقات اللاحقة من المخاطر.

Prompt مقترح للطبقة:

```text
أنت مصنف نصوص ومحدد سياسة ترجمة بين العربية والتركية.

حلل النص قبل الترجمة وحدد نوعه:
- general
- comic
- sacred
- legal
- literary
- marketing

إذا كان المستخدم قد اختار mode يدويًا، احترم اختياره، لكن إذا وجدت جزءًا حساسًا مثل آية أو حديث أو دعاء أو نص قانوني، اذكر ذلك بوضوح.

أخرج:
- detected_mode
- recommended_mode
- has_sacred_segment: true/false
- freedom_level: low/medium/high
- allow_paraphrase: true/false
- sensitive_terms
- warnings

لا تنتج ترجمة نهائية.
```

---

## 15.2 طبقة تحليل النية والسياق

يجب أن تراعي mode:

- إذا كان comic:
  - حلل المتكلم، النبرة، الشخصية، العلاقة بين الشخصيات، والمشهد.
- إذا كان sacred:
  - حلل السياق دون تفسير شرعي زائد.
- إذا كان legal:
  - ركز على الأطراف والالتزامات والقيود.

---

## 15.3 طبقة تحليل المعنى والمفردات

يجب أن تراعي mode:

- إذا كان comic:
  - استخرج العبارات الحوارية، السخرية، التهديد، المزاح، المؤثرات الصوتية.
- إذا كان sacred:
  - استخرج المصطلحات الشرعية ولا تسمح باستبدالها بتعابير عامة.
- إذا كان legal:
  - استخرج المصطلحات القانونية والقيود والاستثناءات.

---

## 15.4 طبقة النحو والصرف

يجب أن تراعي mode:

- إذا كان comic:
  - لا تجعل التحليل النحوي يدفع الترجمة إلى أسلوب جامد.
- إذا كان sacred/legal:
  - انتبه للضمائر، الاستثناءات، النفي، الشروط، القيود، والعموم والخصوص.

---

## 15.5 طبقة الثقافة والتعبيرات الاصطلاحية

هذه الطبقة مهمة وخطيرة.

يجب أن تنص صراحة على:

```text
إذا كان mode = comic:
- ركز على طبيعية الحوار.
- اجعل العبارة مناسبة لفقاعة كلام.
- يجوز التكييف الثقافي الخفيف إذا لم يغيّر المعنى.
- لا تشرح النكتة داخل الترجمة.

إذا كان mode = sacred:
- ممنوع التكييف الثقافي الحر.
- ممنوع إعادة الصياغة التفسيرية.
- ممنوع إدخال تفسير داخل الترجمة.
- حافظ على المصطلحات الشرعية.

إذا كان mode = legal:
- لا تقم بتبسيط قانوني يغير القوة القانونية.
- لا تحذف القيود أو الاستثناءات.

إذا كان mode = marketing:
- يجوز تحسين الأسلوب بشرط عدم تغيير الوعد أو السعر أو المزايا.

إذا كان mode = literary:
- حافظ على الأثر الأدبي دون اختراع معنى جديد.
```

---

## 15.6 طبقة إنتاج الترجمة الأولية

يجب أن تنتج ترجمة حسب mode:

- comic: ترجمة قصيرة، حوارية، طبيعية.
- sacred: ترجمة أمينة وقريبة من الأصل.
- legal: ترجمة دقيقة وغير مختصرة.
- marketing: ترجمة مقنعة دون إضافة وعود.
- literary: ترجمة أدبية دون خيانة المعنى.

---

## 15.7 طبقة المراجعة اللغوية

يجب أن تراجع حسب mode:

- comic:
  - هل الجملة طبيعية؟
  - هل تصلح داخل فقاعة كلام؟
  - هل النبرة محفوظة؟
  - هل الحوار غير رسمي عندما يجب أن يكون غير رسمي؟

- sacred:
  - هل حدث تفسير زائد؟
  - هل تغير مصطلح شرعي؟
  - هل حُذف قيد أو استثناء؟
  - هل الترجمة جميلة لكنها أقل أمانة؟

- legal:
  - هل تغير الالتزام؟
  - هل حُذف شرط أو قيد؟
  - هل المصطلح القانوني مناسب؟

---

## 15.8 طبقة الحكم النهائي

يجب أن تخرج دائمًا:

```text
FINAL_TRANSLATION:
...

BRIEF_REASON:
...

WARNINGS:
...
```

ولا تضف بدائل إلا عند الضرورة القصوى.

في sacred/legal، يجب أن تتحقق من عدم حدوث semantic drift.

في comic، يجب أن تتحقق من أن الترجمة ليست ثقيلة أو مدرسية.

---

# 16. مسارات الترجمة المقترحة

لا يجب استخدام 7 أو 8 طبقات لكل نص دائمًا.

## 16.1 Comic Fast Path

للنصوص القصيرة من الكوميكس:

```text
1. تحليل النبرة والشخصية
2. ترجمة حوارية طبيعية
3. مراجعة قصر الجملة ومناسبة الفقاعة
```

هذا المسار أسرع وأرخص وأفضل للكوميكس القصير.

## 16.2 Sacred Accurate Path

للنصوص الشرعية:

```text
1. تصنيف النص وسياسة الترجمة
2. تحليل السياق
3. تحليل المصطلحات الشرعية
4. تحليل النحو والقيود
5. ترجمة أمينة
6. مراجعة دقة
7. حكم نهائي
```

## 16.3 General Path

للنصوص العامة:

```text
1. تحليل سياق
2. تحليل مفردات
3. ترجمة أولية
4. مراجعة
5. حكم نهائي
```

# 17. أمثلة مهمة

## 17.1 مثال كوميكس

النص:

```text
Are you kidding me?!
```

ترجمة تركية طبيعية للكوميكس:

```text
Dalga mı geçiyorsun?!
```

ليست:

```text
Benimle şaka mı yapıyorsun?
```

الثانية صحيحة لغويًا لكنها أقل حيوية في الكوميكس.

---

## 17.2 مثال شرعي

النص:

```text
أيام العشر
```

الأفضل:

```text
Zilhicce'nin ilk on günü
```

وليس:

```text
Hac ayının ilk on günü
```

لأن الأولى أدق.

---

## 17.3 مثال شرعي آخر

النص:

```text
فلم يرجع من ذلك بشيء
```

ترجمة أقرب:

```text
Bunlardan hiçbir şeyle geri dönmezse
```

لا تجعلها داخل الترجمة الأساسية:

```text
canını ve malını feda etti
```

لأنها تفسيرية أكثر من كونها ترجمة مباشرة.

# 18. تعليمات للوكيل البرمجي Codex / Claude Code / Cursor

استخدم هذا الطلب عند تنفيذ الوثيقة داخل المشروع:

```text
اقرأ الوثيقة:
translation_prompt_spec.md

المطلوب:
1. استخرج جميع System Prompts من الوثيقة.
2. أنشئ ملفًا جديدًا باسم:
   app/prompts/translation_prompts.py
   وإذا كان هيكل المشروع مختلفًا، اختر المكان الأنسب داخل مجلد المشروع.
3. ضع البرومبتات كـ Python constants:
   - BASE_SYSTEM_PROMPT
   - COMIC_SYSTEM_PROMPT
   - SACRED_SYSTEM_PROMPT
   - LEGAL_SYSTEM_PROMPT
   - MARKETING_SYSTEM_PROMPT
   - LITERARY_SYSTEM_PROMPT
4. أنشئ دالة:
   build_system_prompt(mode: str, has_sacred_segment: bool = False) -> str
5. أنشئ دالة:
   mode_label(mode: str) -> str
6. عدّل كود translation pipeline بحيث يستخدم build_system_prompt حسب mode.
7. لا تضع كل البرومبتات في كل طلب.
8. BASE_SYSTEM_PROMPT يستخدم دائمًا.
9. البرومبت المتخصص يضاف فقط حسب mode.
10. إذا كان mode = comic لكن النص يحتوي آية/حديث/دعاء، أضف SACRED_SYSTEM_PROMPT كقواعد حماية.
11. حافظ على نظام الطبقات الحالي ولا تكسره.
12. أضف طبقة "تصنيف النص وسياسة الترجمة" قبل الطبقات الحالية.
13. عدّل طبقة الثقافة والتعبيرات الاصطلاحية حتى لا تطبق التكييف الحر على sacred/legal.
14. بعد التعديل، أعطني ملخصًا للملفات التي تغيرت.
```

---

# 19. ملاحظات تصميم مهمة

- لا تعتمد على LLM فقط لاكتشاف النصوص الشرعية. الأفضل لاحقًا إضافة قواعد مساعدة مثل Regex وقاموس مصطلحات.
- لا تجعل كل النصوص تمر على كل الطبقات؛ هذا مكلف وبطيء.
- الكوميكس يحتاج سرعة وطبيعية أكثر من التحليل العميق.
- النصوص الشرعية والقانونية تحتاج أمانة أكثر من الطلاقة.
- أكبر خطر في المشروع هو semantic drift: أن تظل الترجمة جميلة لكنها تنحرف عن المعنى الأصلي.
- أكبر فرصة في المشروع هي دعم الكوميكس العربي/التركي بشكل طبيعي مع حماية ذكية للنصوص الحساسة.

# 20. الخلاصة التنفيذية

المطلوب من المشروع أن يصبح عنده مترجم يغير سلوكه حسب نوع النص:

- comic: مترجم حواري سريع وطبيعي.
- sacred: مترجم أمين ومحافظ.
- legal: مترجم دقيق وغير مختصر.
- marketing: مترجم مقنع دون اختراع وعود.
- literary: مترجم يحافظ على الجمال دون خيانة المعنى.
- general: مترجم طبيعي ومتوازن.

لا تستخدم Prompt واحد ضخم دائمًا. استخدم Base Prompt دائمًا، ثم أضف Prompt متخصص حسب mode.
